# Tasks

Current backlog. One task per line.

Format:

```
description | priority | due date | goal
```

- **priority** — P0 (urgent), P1 (important), P2 (when I can). Required.
- **due date** — YYYY-MM-DD. Optional. Leave blank if no deadline.
- **goal** — a short tag matching a goal in `goals.md` (e.g. conversion, onboarding, discovery). Optional.

Keep the list sorted by due date ascending. Undated tasks go at the bottom. Within the same date, order by priority (P0 first).

Add tasks with `/new-task`. Delete lines when done, or move them to an archive during weekly review.
