# Improvements

Running log of things to fix in the personal operating system — ideas that would make the system smarter, tighter, or less annoying.

Format:

```
problem | priority | date added
```

- **priority** — P0 (major friction, something broke), P1 (useful fix), P2 (nice to have)
- **date added** — YYYY-MM-DD

Add entries manually as you notice friction, or let `/weekly-review` log them automatically from the past week's sessions. Work through them with `/fix-one-thing`.

Keep the list sorted by priority (P0 first), then by date added (oldest first — older items have been waiting longer).
